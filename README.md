# 文件收纳箱 FileAggregator

> 纯本地离线 · 零拷贝映射 · 一站式管理你的视频/图片/文档

一个 100% 本地运行的桌面文件聚合软件：拖一个文件夹进来，自动按 **视频 / 图片 / 文档** 三大类整齐展示，全程只做"路径引用"，**绝不复制、不移动、不修改、不删除你电脑上的任何原始文件**。

---

## 功能特性

| 模块 | 能力 |
|------|------|
| 收录 | 拖拽 / 选择按钮添加任意磁盘的文件夹，支持 C/D/E/F 任意盘 |
| 扫描 | 自动递归扫描，按视频/图片/文档三类聚合 |
| 视频 | 内置 HTML5 播放器（mp4/webm/mov 等），其他格式调用系统播放器；**悬浮预览**（OpenCV 帧序列循环播放） |
| 图片 | 内置大图预览，缩略图懒加载，支持 jpg/png/gif/bmp/webp 等；**左右箭头键跨文件夹切换** |
| 文档 | 一键调用系统默认软件打开（Word/Excel/PDF 阅读器等） |
| 项目 | **文件专区（项目专区）**：以文件夹为单位管理，不遍历内部文件，支持收藏/删除/定位/批量操作 |
| 定位 | 一键打开任意文件的"所在位置"（资源管理器并选中） |
| 刷新 | 单目录刷新 / 全部刷新两种模式 |
| 排序 | **名称 / 修改时间 / 创建时间 / 大小** 四种排序，升序/降序一键切换 |
| 分页 | **10/20/50/100 条/页** 可选，智能分页导航 |
| 收藏 | **收藏夹体系**：视频/图片/文档/**项目** 四个专区独立收藏，支持新建/重命名/删除收藏夹 |
| 搬迁 | **收藏夹一键搬迁**：批量剪切移动收藏夹内所有真实文件到统一文件夹，自动同步路径 |
| 删除 | **双模式删除**：普通移除（仅删软件记录，无确认）/ **永久删除**（高危操作，二次确认） |
| 存储 | 配置永久保存到 `~/.file_aggregator/config.json`，重启不丢 |
| 安全 | 全程只读 + 路径映射，绝不动用户原文件 |

## 开发期运行

### 1. 安装依赖

```bash
pip install pywebview opencv-python
```

`pywebview` 用于桌面窗口，`opencv-python` 用于视频封面截取与悬浮预览帧提取。其余 `os/json/subprocess/base64` 都是 Python 标准库。

### 2. 启动

```bash
python main.py
```

第一次启动若提示需要 WebView2 Runtime，Win10 1809+ / Win11 通常已自带；若没有，按系统提示一键安装即可（**Microsoft 官方组件，纯本地**）。

## 一键打包成单文件 EXE

### 方式 A：双击 `build.bat`（推荐）

直接双击项目根目录的 `build.bat`，脚本会：

1. 检查 Python 环境
2. 自动安装 `pywebview` + `pyinstaller` + `opencv-python`
3. 清理旧构建产物
4. 执行打包
5. 完成后自动打开 `dist` 目录

产物：**`dist/文件收纳箱.exe`** —— 单文件、自带运行时、双击即用。

### 方式 B：手动一键命令

```bash
pip install pywebview pyinstaller opencv-python && pyinstaller --onefile --noconsole --clean --name "文件收纳箱" --add-data "ui.html;." --collect-all webview --collect-all cv2 --hidden-import webview.platforms.edgechromium --hidden-import webview.platforms.mshtml --hidden-import webview.platforms.winforms main.py
```

| 参数 | 作用 |
|------|------|
| `--onefile` | 打包成单个 exe 文件 |
| `--noconsole` | 不显示后台 cmd 黑窗 |
| `--clean` | 清理 PyInstaller 缓存 |
| `--name` | 指定 exe 文件名 |
| `--add-data "ui.html;\."` | 把前端 HTML 一起塞进 exe |
| `--collect-all webview` | 收集 pywebview 的所有资源 |
| `--collect-all cv2` | 收集 OpenCV 的所有资源 |
| `--hidden-import xxx` | 显式包含 pywebview 各后端模块 |

打包完成后，`dist/文件收纳箱.exe` 即为最终产物，**自带完整 Python 运行环境**，可直接发给任何 Win10/Win11 用户使用，对方无需安装 Python / 任何依赖。

## 数据存储位置

所有收录的目录路径、收藏夹、排序设置都存放在：

```
C:\Users\<你的用户名>\.file_aggregator\config.json
```

文件内容形如：

```json
{
  "folders": [
    "D:\\我的视频",
    "E:\\照片备份\\2024",
    "F:\\工作文档"
  ],
  "project_folders": [
    "D:\\项目A",
    "E:\\设计稿合集"
  ],
  "favorites": {
    "video": [{"id": "fv_xxx", "name": "精彩瞬间", "items": ["D:\\我的视频\\1.mp4"]}],
    "image": [],
    "document": [],
    "project": []
  },
  "settings": {
    "sort_by": "name",
    "sort_order": "asc",
    "page_size": 20
  }
}
```

- 想清空所有收录记录：删除该 `config.json` 即可
- 想迁移到另一台电脑：复制这个 json 过去同名位置
- **再次强调**：删除 json 只会清空软件内的索引，**不影响磁盘上的任何真实文件**

## 安全承诺（绝不触碰原文件）

代码层面已严格遵守"只映射不复制"原则：

| 操作 | 实现方式 | 是否动原文件 |
|------|----------|-------------|
| 添加目录 | 把绝对路径字符串 append 进 JSON 数组 | 否 |
| 添加项目目录 | 把绝对路径字符串 append 进 `project_folders` 数组 | 否 |
| 扫描目录 | `os.walk()` 只遍历不写入 | 否 |
| 缩略图 | `open(path, 'rb').read()` 只读 | 否 |
| 打开文件 | `os.startfile(path)` 调用系统程序 | 否 |
| 定位文件 | `explorer /select` 调用资源管理器 | 否 |
| 移除目录 | 从 JSON 数组中 remove 路径字符串 | 否 |
| 移除项目目录 | 从 `project_folders` 数组中 remove 路径字符串 | 否 |
| 收藏文件 | 在 JSON 中保存路径字符串 | 否 |
| 一键搬迁 | `shutil.move(path)` 批量剪切移动原文件（高危二次确认） | 是 |
| 永久删除 | `shutil.rmtree(path)` 真实删除（高危二次确认） | 是 |

除"永久删除"和"一键搬迁"外，整个项目里没有任何 `shutil.copy / shutil.move / os.remove / open(..., 'w')` 用在原文件路径上。

## 常见问题

**Q1：拖拽不识别？**
A：某些精简版 WebView 拿不到 `file.path`。请改用顶部 "选择目录" 按钮，效果完全一致。

**Q2：双击 mkv / rmvb 没反应？**
A：浏览器内核原生不支持这些格式，软件会自动调用系统默认播放器（如 PotPlayer / VLC）。

**Q3：大视频 (>500MB) 预览失败？**
A：内置播放器通过 base64 加载，太大会爆内存，已限制 500MB；超过的请用系统播放器。

**Q4：在朋友的电脑能跑吗？**
A：能。`build.bat` 打出的 exe 自带 Python 运行时，仅依赖 Win10/11 系统自带的 WebView2 Runtime。99% 的新电脑都开箱即用。

**Q5：能不能改成跨平台？**
A：代码本身兼容 macOS / Linux（`subprocess` 的 `open`/`xdg-open` 已写好），但 `build.bat` 仅 Windows；其他平台用 `pyinstaller main.py` 自行打包即可。

**Q6：收藏夹里的文件移动/重命名了怎么办？**
A：收藏夹只保存文件路径字符串。若原文件被移动或重命名，路径失效，收藏夹中该条目会被静默跳过。建议定期整理文件后重新收藏。
